# Quick Start Guide

This guide will help you quickly get up and running with Noah Template for development.

## Prerequisites

Ensure you have:
- Node.js (v18+)
- pnpm (v8+)
- PostgreSQL database access

## Five-Minute Setup

<Stepper>
  <StepperItem title="Clone and Install">
    ```bash
    # Clone the repository
    git clone https://github.com/yourusername/noah-template.git
    cd noah-template
    
    # Install dependencies
    pnpm install
    ```
  </StepperItem>

  <StepperItem title="Configure Environment">
    Create a `.env` file in the `apps/api` directory:
    
    ```bash
    # Database
    DATABASE_URL=postgresql://username:password@localhost:5432/dbname
    
    # Auth
    BETTER_AUTH_SECRET=your-secret-here
    BETTER_AUTH_URL=http://localhost:8787
    ALLOWED_ORIGINS=http://localhost:3000,http://localhost:8787
    ```
  </StepperItem>

  <StepperItem title="Start Development Servers">
    ```bash
    # Start all services
    pnpm dev
    ```
    
    This will start:
    - Web app on http://localhost:3000
    - API on http://localhost:8787
    - Docs on http://localhost:3001
  </StepperItem>
</Stepper>

## Key Development Commands

| Command | Description |
|---------|-------------|
| `pnpm dev` | Start all development servers |
| `pnpm build` | Build all packages and applications |
| `pnpm lint` | Run linting on all packages |
| `pnpm -F api dev` | Start just the API server |
| `pnpm -F web dev` | Start just the web application |
| `pnpm -F docs dev` | Start just the documentation site |

## Database Commands

```bash
# Push schema changes to database
pnpm -F @repo/db db:push

# Generate migrations
pnpm -F @repo/db db:generate

# Run migrations
pnpm -F @repo/db db:migrate

# Open Drizzle Studio
pnpm -F @repo/db db:studio
```

## Authentication Flow

The template includes a complete authentication system using Better Auth:

1. User visits the sign-in page on the web app
2. Authentication requests are sent to the API
3. API validates credentials and creates a session
4. Session cookie is returned to the web app

<Note title="Auth Endpoints">
  The authentication API is available at `/api/auth/*` on the API server.
  The web app is pre-configured to connect to these endpoints.
</Note>

## Troubleshooting

If you encounter issues:

- Ensure all environment variables are correctly set
- Check that PostgreSQL is running and accessible
- Make sure all packages are installed with `pnpm install`
- Try restarting the development servers with `pnpm dev`
- Check the console for specific error messages
